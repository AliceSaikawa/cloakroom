# claude-code-pii — モデル運用ルール

Claude Code v2.1.88 デコンパイルソース上のPIIフィルタフォーク。グローバル `~/.claude/CLAUDE.md` の委譲基準に、本プロジェクトでは以下を上乗せする。

## モデル階層（このフォルダの原則）

```
Fable 5   … 上流工程のみ（アーキテクチャ判断・設計。実装/調査/レビューはしない）
Opus 4.8  … 難度の高いタスク（高難度デバッグ、微妙なトレードオフ判断、メインセッションのデフォルト）
Sonnet 5  … 通常タスク（探索、量産実装、翻訳、セキュリティ監査）
Haiku 4.5 … 機械的タスク（ビルド修正、テスト実行・整形）
```

原則: **Fable起動前に下位モデルで机を整える**（調査は explorer、失敗要約は test-runner を先に走らせ、Fableには圧縮サマリーだけ渡す）。

## エージェントルーティング（.claude/agents/）

| Agent | Model | 用途 |
|-------|-------|------|
| explorer | sonnet | ソース探索・データフロー調査（Fable/Opus起動前の下準備） |
| implementer | sonnet | 確定済み設計の実装、テスト追加、4言語ドキュメント同期 |
| pii-auditor | sonnet | PII漏洩経路の監査（フィルタ変更後に必須） |
| hard-debugger | opus | Sonnetが2回失敗した問題、minified名の挙動解明 |
| build-fixer | haiku | ビルド・型エラーの最小diff修正 |
| test-runner | haiku | test-*.mjs 実行と失敗要約（修正はしない） |

- エージェントは圧縮サマリーのみ返す（最大30行、生コード・全diff禁止）
- エスカレーション: sonnet 2回失敗 → hard-debugger (opus) → それでも設計問題ならメインをFableへ

## メインセッションの切替ルール

メインのモデルはClaude自身では変更できないため、Claudeは**提案まで**を担当し、ユーザーが `/model` で切り替える。

- デフォルト: **Opus 4.8**
- **「Fableに切り替え推奨」と提案する条件**（上流工程のみ）:
  - マスク位置・復元戦略・プレースホルダ設計などフィルタアーキテクチャの変更判断
  - cli.js 全体のデータフロー俯瞰が必要で、誤ると PII 漏洩に直結する判断
  - hard-debugger (opus) でも解けず設計問題と判明した時
- Fableでの判断が完了したら「Opusに戻してOK」と明示する
- Fable稼働中はRead/Grepを最小化し、収集・整形はサブエージェントに委譲する

## 単発・未確定業務

繰り返すと確認できるまで定義ファイル化しない。上の階層原則に沿ってその場で振り分け、2回以上繰り返したらエージェント定義に昇格させる。
